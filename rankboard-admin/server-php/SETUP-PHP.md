# RankBoard PHP backend (Laravel) — setup

This folder (`server-php-overlay/`) contains **RankBoard's application
code** for Laravel: controllers, models, services, routes, migration,
seeder, and config. It is an **overlay**: you create a fresh Laravel
skeleton with Composer, then copy these files on top of it.

Why an overlay instead of a complete folder? A Laravel project's
skeleton (the `vendor/` directory and framework boilerplate) is never
shipped by hand — it's generated by Composer on your machine, the same
way `node_modules` is generated by npm. The overlay holds only what's
ours.

The React client needs **zero changes**. It talks to the same
`/api/...` contract, so once this server is running on port 4000, the
same UI works with a PHP brain instead of a Python one.

---

## 0. One-time: install PHP + Composer

Easiest path on Windows (and Mac): install **Laravel Herd** from
https://herd.laravel.com — one installer that gives you PHP, Composer,
and the `laravel` command. After installing, **open a new terminal**
and check:

```
php -v
composer -V
```

Both should print versions. (Alternative: install PHP from php.net and
Composer from getcomposer.org manually, but Herd is much less fiddly.)

## 1. Create the Laravel skeleton

From the **project root** (the folder with `client/`, `server-python/`,
`package.json`):

```
composer create-project laravel/laravel server-php
```

This downloads a fresh Laravel app into a new `server-php/` folder
(takes a minute). Laravel uses **SQLite by default** — same as our
other backends, no database server to install.

## 2. Add the API layer + Excel library

```
cd server-php
php artisan install:api
composer require phpoffice/phpspreadsheet
cd ..
```

`install:api` adds **Sanctum** (the token auth our login uses) and
creates `routes/api.php`. If it asks *"Would you like to run your
database migrations?"* answer **No** for now — we migrate after the
overlay is in place. `phpspreadsheet` is the Excel read/write library
(PHP's equivalent of Python's openpyxl).

## 3. Copy the overlay onto the skeleton

From the project root —

Windows (Command Prompt or PowerShell):

```
xcopy server-php-overlay server-php /E /Y
```

Mac/Linux:

```
cp -R server-php-overlay/. server-php/
```

This intentionally **overwrites three skeleton files**:
`bootstrap/app.php`, `routes/api.php`, and
`database/seeders/DatabaseSeeder.php`. Everything else is added new.

## 4. Configure

Open `server-php/.env` and add these lines at the bottom (same values
you use in `server-python/.env`):

```
APP_URL_FRONTEND=http://localhost:5173

DATAFORSEO_LOGIN=you@example.com
DATAFORSEO_PASSWORD=your_api_password
RANK_LOCATION_CODE=2036

# Optional:
# DATAFORSEO_BASE=https://sandbox.dataforseo.com
# RANK_LANGUAGE=en
# RANK_CHECK_DEPTH=30
# RESEND_API_KEY=
# EMAIL_FROM=RankBoard <onboarding@resend.dev>
```

Leave the DataForSEO lines out (or empty) to run in free simulated
mode, exactly like the other backends.

## 5. Create + seed the database

```
cd server-php
php artisan migrate:fresh --seed
```

You should see the migrations run and then:
`Seeded first Super Admin -> soham@infyappdevelopment.com / admin123`.

## 6. Run it

```
php artisan serve --port=4000
```

Port **4000** matters — that's where the React dev server forwards
`/api` calls. In a **second terminal**, from the project root:

```
npm run dev --prefix client
```

Open http://localhost:5173, sign in as `soham@infyappdevelopment.com` /
`admin123`. Same app, PHP backend. (Or use the shortcut from the
project root: `npm run dev:php` runs both at once.)

> Only ONE backend can hold port 4000 at a time — stop the Python or
> Node server first.

Note each backend keeps its **own database file** (this one lives at
`server-php/database/database.sqlite`), so people/projects you create
under one backend won't appear under another.

## Troubleshooting

- **`php` or `composer` is not recognized** → Herd isn't on your PATH
  yet; close and reopen the terminal (or restart) after installing.
- **`Class "PhpOffice\PhpSpreadsheet\..." not found`** → step 2's
  `composer require phpoffice/phpspreadsheet` was skipped.
- **Every request returns 401 / `auth:sanctum` errors** → step 2's
  `php artisan install:api` was skipped, so Sanctum is missing.
- **`Base table or view not found`** → step 5 (`migrate:fresh --seed`)
  hasn't been run.
- **Port 4000 already in use** → the Python/Node API is still running;
  stop it (Ctrl+C in its terminal) and start this one.
- **Changed `.env` but nothing happened** → restart `php artisan serve`;
  and if you ever ran `php artisan config:cache`, run
  `php artisan config:clear`.
